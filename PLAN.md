# مخطط "شغلني" - النسخة النهائية القطعية

> بعد مناقشة كاملة لجميع النقاط. هذا هو المخطط النهائي.

---

## 1. مفهوم التطبيق

منصة وسيطة تربط أصحاب المهارات بأصحاب العمل في اليمن لكل أنواع الخدمات (يدوية، تقنية، تعليمية، منزلية، وغيرها). نظام طلبات وعروض مع دفع آمن (Escrow) وتواصل داخلي بدون أرقام.

---

## 2. المستخدمون

| الخاصية | القرار |
|---|---|
| الحساب | **حساب واحد** - نفس المستخدم يقدر يكون صاحب مهارة وصاحب عمل |
| مستويات المستخدمين | للـ MVP: جديد / موثّق. مستقبلاً: خوارزميات ظهور ورسوم ملفات مميزة |
| التسجيل | أي شخص برقم جوال يمني، بدون شروط |
| التوثيق | صاحب المهارة: رقم جوال + اسم كامل + صورة هوية |
|  | صاحب العمل: رقم جوال + اسم كامل |
| ظهور الأرقام | **ممنوع تماماً** - التواصل كله داخل التطبيق |

---

## 3. الخدمات والطلبات

| الخاصية | القرار |
|---|---|
| صاحب المهارة | يقدر **ينشر إعلان خدمة** + **يقدم عروض على طلبات** (أو وحدة فقط) |
| صاحب العمل | يقدر **ينشر طلب** + **يتصفح إعلانات الخدمات ويطلب مباشرة** |
| الصور | اختيارية لكنها تظهر أولاً في النتائج |
| فئات الخدمات | مجال مفتوح - صاحب المهارة يكتب فئة خدمته مع مراجعة من الأدمن لإضافة فئات جديدة |

---

## 4. آلية الصفقة (Workflow)

```
1. صاحب العمل ينشر طلب (أو يتصفح إعلانات)
2. أصحاب المهارات يقدمون عروض (سعر + مدة + وصف)
3. صاحب العمل يختار العرض يدوياً
4. شاشة تأكيد (سعر، مدة، وصف، نسبة المنصة)
5. صاحب العمل يوافق ويدفع (كامل أو على دفعات حسب الاتفاق)
6. المبلغ محجوز في المنصة (Escrow)
7. صاحب المهارة يبدأ وينفذ
8. صاحب العمل يؤكد الاستلام (أو يفتح نزاع خلال 3 أيام)
9. المنصة تصرف المبلغ لصاحب المهارة فوراً
10. تقييم متبادل بين الطرفين
```

---

## 5. الدفع

| الخاصية | القرار |
|---|---|
| طريقة الدفع | **كامل أو دفعات** - حسب اتفاق الطرفين |
| طريقة الشحن | **خيارين:** يشحن محفظته أولاً أو يدفع مباشر لكل صفقة |
| طرق الشحن المتاحة | كريمي - جيب - ون كاش - جوالي - صرافين معتمدين |
| السحب | **فوري** بعد تأكيد صاحب العمل |
| أقل مبلغ للسحب | يحدد لاحقاً حسب تكاليف التحويل |

---

## 6. نسبة المنصة (بعد أول 6 شهور مجانية)

نسب متفاوتة حسب الخدمة - تحدد لاحقاً بناءً على تحليل التكاليف.

النموذج المبدئي المقترح:
- الخدمات التقنية: 10%
- الخدمات اليدوية: 7%
- الخدمات التعليمية: 5%
- الخدمات المنزلية: 7%

---

## 7. النزاعات

| الخاصية | القرار |
|---|---|
| مين يبدأ نزاع | **الطرفين** |
| مهلة النزاع بعد التأكيد | **3 أيام** |
| الأدلة المقبولة | صور، محادثات، ملفات (مسجلة داخل التطبيق فقط) |
| الحكم | فريق المنصة (غير قابل للاستئناف في المرحلة الأولى) |
| القرارات الممكنة | كامل المبلغ لصاحب المهارة / استرداد كامل / تعويض جزئي |

---

## 8. التقييم

| الخاصية | القرار |
|---|---|
| من يقيم مين | **تقييم متبادل** (الطرفين يقيمون بعض) |
| النظام | 5 نجوم + تعليق كتابي |
| الظهور | فوري بعد التقييم |

---

## 9. الإشعارات

كل الأحداث ترسل إشعار داخل التطبيق:
- طلب جديد في مجال الخدمة
- عرض سعر على طلبي
- قبول/رفض عرضي
- تم الدفع - ابدأ الشغل
- تم التسليم - راجع
- تذكير بعدم التأكيد (بعد 3 أيام)
- تقييم جديد

---

## 10. البحث والتصفح

- بحث نصي + فلاتر (مدينة، فئة، سعر، تقييم)
- أقسام للتصفح بدون بحث
- مستقبلاً: بحث صوتي

---

## 11. خطة الإطلاق

| المرحلة | المدة | الوصف |
|---|---|---|
| **MVP (ويب + تطبيق جوال)** | 6-8 أسابيع | React Native (Android + iOS) + Web (PWA) |
| **Closed Beta** | أسبوعان | 50-100 مستخدم مدعو، اختبار وتعديل |
| **Open Beta** | مستمر | فتح للكل + تسويق عبر مشهورين |
| **التسجيل القانوني** | من البداية | تسجيل شركة رسمية في اليمن |

---

## 12. التقنية (مؤكدة)

| الطبقة | التقنية | السبب |
|---|---|---|
| **Backend API** | Node.js + Express | سريع، مناسب لـ Vibe Coder، مساعدات AI كثيرة |
| **Database** | PostgreSQL | علاقاتي، قوي، مجاني |
| **Mobile App** | React Native (Expo) | كود واحد Android + iOS |
| **Admin Panel** | React + Tailwind (ويب) | خفيف وسريع |
| **Authentication** | JWT + OTP (رقم جوال) | أمان وبساطة |

## 14. شروط الاستخدام (بنود أساسية)

- استخدام التطبيق يعني الموافقة على تحكيم المنصة في النزاعات
- أي اتفاق خارج التطبيق ليس مسؤولية المنصة
- المنصة غير مسؤولة عن جودة الخدمة (التقييمات هي المرجع)
- يحق للمنصة تجميد أي حساب في حال مخالفة الشروط
- نسبة المنصة قابلة للتغيير بعد الإعلان قبل 30 يوماً
